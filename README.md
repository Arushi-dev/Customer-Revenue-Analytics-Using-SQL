# Customer & Revenue Analytics Using SQL

## Overview

This project analyzes transactional retail data using SQL queries executed through DuckDB to uncover key business insights related to customer behavior, revenue drivers, and product performance.

The goal of the analysis is to demonstrate how SQL can be used to transform raw transactional data into meaningful business insights.

---

## Key Questions

The analysis focuses on answering the following questions:

- What is the total revenue generated by the business?
- Which countries contribute the most revenue?
- Which customers generate the highest value?
- Which products drive the most revenue?
- How does revenue change over time?

---

## Key Insights

- Revenue is highly concentrated among a small group of high-value customers.
- The United Kingdom contributes the majority of total revenue.
- Decorative home products generate the largest share of product revenue.
- Revenue increases toward the end of the year, indicating seasonal demand.
- Filtering return transactions is necessary to accurately calculate sales revenue.

## Project Workflow

The project follows a typical data analytics workflow:

1. Load and prepare the dataset
2. Clean return transactions
3. Compute key business KPIs
4. Analyze revenue by country
5. Identify high-value customers
6. Evaluate product performance
7. Analyze revenue trends over time
